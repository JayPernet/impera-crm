# PRD Master: Impera CRM - Product Requirements Document

This document serves as the central hub for all product requirements for Impera CRM. It links to detailed Epics and User Stories, providing a comprehensive overview of the product's functionality and development roadmap.

## Epics (Phase 1: Core/MVP)

The following Epics represent the major functional blocks planned for the initial Core/MVP phase of the Impera CRM. Each Epic will be further detailed in its respective document, which will contain linked User Stories.

*   [EPC-01-User-Auth-Management](epicos/EPC-01-User-Auth-Management.md)
    *   *Covers: User Management & Authentication, Multi-tenant System Foundation, Role-Based Access Control.*
*   [EPC-02-Lead-Management](epicos/EPC-02-Lead-Management.md)
    *   *Covers: Functionality for creating, viewing, editing, and deleting leads.*
*   [EPC-03-Client-Management](epicos/EPC-03-Client-Management.md)
    *   *Covers: Functionality for creating, viewing, editing, and deleting client records.*
*   [EPC-04-Property-Management](epicos/EPC-04-Property-Management.md)
    *   *Covers: Functionality for adding, viewing, editing, and deleting property listings.*
*   [EPC-05-Dashboard-Overview](epicos/EPC-05-Dashboard-Overview.md)
    *   *Covers: Centralized dashboard displaying key metrics and recent activities.*
*   [EPC-06-WhatsApp-Integration](epicos/EPC-06-WhatsApp-Integration.md)
    *   *Covers: Basic integration for sending and receiving WhatsApp messages linked to leads/clients.*

## Phase 2 & 3 Epics

Epics for Phase 2 (Necessary Features) and Phase 3 (Polish & Scale) will be documented as the project progresses and after the successful completion of the Core/MVP phase.
